Summary

I was the nerdy kid obsessed with the periodic table and Rubik's cube; read a whole book about Rubik’s cube algorithms and spent a week trying to figure out an algorithm to recursively calculate the atomic mass of chemical elements (the latter was less successful 😉). I’m now a Hubble fellow doing research in astrophysics at “the place that made nerd cool.” I enjoy geocaching and can be pretty stubborn about finding a cache once I've got my eye on it. Spent half a holiday solving GC2T824, much to my husband’s despair. 😁

Things many people find weird about me: I type on a DasKeyboard Ultimate (that's a blank keyboard), use a Dvorak layout, sometimes wear funny Vibram FiveFingers, and can't eat with my glasses on.

  • PyXel

    Starting 2016

    PyXel is a Python package for modeling astronomical X-ray imaging data. It provides methods for modeling surface brightness profiles from X-ray satellites using a variety of models and statistics. While various packages have been written for modeling X-ray spectra, the modeling of X-ray images is still being done using private, ad-hoc scripts. PyXel aims to minimize duplicate effort, provide users with a consistent way of analyzing X-ray data from any satellite, and supply clean maintainable code to which everybody may contribute. The code mainly uses the following packages: NumPy, SciPy, Astropy, emcee, corner, and tabulate. I presented an alpha release of PyXel at this year’s American Astronomical Society meeting.

    Features:
    • Surface brightness profiles created using DS9 region files.
    • Linearly- or logarithmically-binned profiles to achieve a minimum number of counts.
    • Models implemented: ß-model, power-law, broken powerlaw, plus all the models available in astropy.modeling.
    • Chi-square and maximum likelihood statistics (extended Cstat—supports Poisson background).
    • Optional model integration over each bin to prevent incorrect best-fitting parameters in the case of discontinuous models such as broken power-laws.
    • Parameter uncertainties evaluated using MCMC.
    • Option to automatically create LaTeX-formatted tables of the best-fitting models.

    Team members:
    • Georgiana Ogrean,
    • Armin Burgmeier
