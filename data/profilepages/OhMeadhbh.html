Summary

I am a software industry veteran with over 20 years experience developing, managing and supporting complex software projects in the mobile, security and virtual worlds industry sectors. As a community leader, I was responsible for spearheading the effort that became the Virtual Worlds Region Agent Protocol (VWRAP) working group in the IETF and for co-founding the Homebrew Mobile Phone Club. As a software architect I designed APIs and implementations for embedded devices such as the Gibson Wurlitzer Digital Jukebox. As a security architect, I have designed secure boot schemes and security services for Handspring, PalmSource and Divitas. As an implementer, I have contributed to the security and interface software found on many mobile phones and portable devices. I have developed web-based interfaces to control GSM Radios on mobile devices as well as security and messaging middleware.

My name is Irish and is pronounced like the english “Maeve.”

I possesses many skills relevant to software development. Chief amongst them my ability to manage and motivate small development teams.. As a software engineer, I am capable of developing for both large systems (elastic clouds) or small systems (mobile devices.) I is experienced with development environments including the GNU toolchain on Unix(tm) and Linux systems and various versions of Microsoft's Vis Dev Studio and Platform Builder. I am experienced with traditional programming languages such as C and assembly as well as more recent OO languages like Java, C++ and Python. My work on embedded web products and protocols has also exposed me to PHP and JavaScript.

As a seasoned software engineer and engineering manager, I am experienced with several development methodologies. I ha e coached teams through the process of organizational change and adoption of Scrum and XP. My experience with software and information security allows me to supplement my engineering management coaching with software risk management practices.

Specialties: Software Development Management, Lean Software Development, Extreme Programming, Agile Programming Methodologies, Software Project Management, Software Product Estimation, Software Development Organization Formation, Management, Smalltalk, Java, C++, Embedded System Development, Information Security

  • Software Architect

    Linden Lab
    April 2008 – February 2010 (1 year 11 months)

    At Linden Research, I participated in the development of the "next generation" protocols and server architecture for Second Life(tm). Intended to enable the flow of information between the "2d web" and emerging open virtual worlds, these protocols were eventually published as the "Open Grid Protocol (OGP)" suite. As interest in virtual worlds grew, I led a collaborative effort between Linden Research, Intel, IBM and several independent open source software developers to form the Virtual Worlds Region Agent Protocol (VWRAP) working group in the IETF. I am the author, co-author or editor of most of this working group's output.

    My duties included re-architecting the existing Second Life server code to adhere to open standards including OAuth, COLLADA, XMPP and the VWRAP suite. I was also responsible for managing the VWRAP-enabled "test grid," used by open source developers to interact with a small, private virtual world for testing purposes. Development was in JavaScript / Node.JS, Python and C++ on Debian Linux.

  • Chief Cat Herder

    Homebrew Mobile Phone Club
    March 2006 – April 2008 (2 years 2 months)

    In 2006, I co-founded the Silicon Valley Homebrew Mobile Phone Club. Comprised of developers and entrepreneurs from mobile and traditional technology value-chains, the club's mission was to accelerate innovation in the mobile handset space by developing a "complete open phone." Under the name "Matthew Hamrick", I penned a number of articles describing how innovation in the mobile "intangibles economy" is best served by eliminating the cost of proprietary software development environments and licensed hardware designs.

    The work of the club culminated in the demonstration of the "rPhone" at the 2007 Maker Faire. Running an embedded Linux with a custom web-based interface, the rPhone utilized only open source software, generally available components, available in small or single quantities with circuit board and enclosure designs released under creative commons attribution-only licenses. The rPhone was able to send and receive text messages and cellular calls.

    Developed the "Jowles" telephony UI consisting of a C/C++ based GSM control server communicating via HTTP to a SVG / JavaScript UI rendered remotely in a browser or locally directly to the frame buffer.

  • Software Security Architect

    DiVitas Networks
    December 2006 – December 2007 (1 year 1 month)

    Given the mandate to "make our product better! make it secure!" I took a three-pronged approach: improve the code, improve the process and educate the coders. With other members of the development team I led a security evaluation of the existing code and architecture artifacts, looking for known vulnerabilities.

    As currently defined, SRTP (or Secure Real-Time Protocol) specifies the use of the AES cipher. This cipher is computationally intensive for some mobile devices; to avoid this problem, I developed an extension to SRTP to support the HC-128 and HC-256 ciphers.

    Coding was mostly in C/C++ on Linux with some web interface programming in JavaScript.

  • Software Security Architect

    PalmSource, Inc.

I was also responsible for advising management regarding the Software Security Development Lifecycle and security incident planning. I also developed security policy enforcement standards for the LiMo Foundation.

    My work on the SPF can be downloaded as part of the Access Linux Platform's Hiker Project. The SPF itself is written in C/C++ and implemented as an enhancement for DBUS, a kernel-space Linux Security Module (LSM) and a user-space security policy daemon.

Intended to secure inter-departmental electronic communications, the "Fed Bridge" project required detailed evaluation of commercial security products and specification for how diverse collections of products could be effectively concerted to protect sensitive information assets.

    Following my work with DHS, I focused on delivering cost-effective information processing solutions to government and quasi-government agencies using Commercial Off The Shelf (COTS) products and open source components. Specializing in building systems by integrating commercial "Software as a Service" and "Web 2.0" components, I was able to deliver flexible solutions to federal customers on time and under budget.

    I used Java-based web and application servers extensively during this period including WebSphere, JBOSS, Tomcat, Struts, Hibernate, Servlets and JSPs.

As manager of this group, I was responsible for hiring / firing decisions, promotions and employee development, budgeting, task design, export control licensing, visa processing and negotiating supported features with product management.

I also participated in the development of an early "IIOP over SSL" solution using the Consensus SSL library, the RSA B/SAFE crypto toolkit and VisiBroker CORBA ORB.

In conjunction with the product management team I developed a secure management interface for the Rapport Dialup Switch. This interface applied and verified a keyed message authentication code to each SNMP message generated and accepted by the switch. Key material was generated centrally and distributed using the key escrow facility of the Entrust middleware.

    Code was written in C and used the human interface facilities of HP's OpenView system management infrastructure. The implementation of the code was used to demonstrate both the use of the Entrust toolkit and the SNMP trap structure for the Rapport Dialup Switch product line.
