Projects

  • Brightcove

    June 2016 – September 2016

    Brightcove was looking to add an oft-requested feature of their customers: the ability to do some basic trimming of videos, optionally stitch pre-roll and/or post-roll bumpers, and publish it as a new video. We leveraged ffmpeg's command-line interface to achieve what we needed, and by using the -filter-complex argument, we were able to achieve any combination of the above in a single command. Doing multiple passes in ffmpeg brings the quality way down, and we couldn't have that. The code touched both the UI and leveraged a brand new EC2 worker to do the heavy lifting with ffmpeg. It was all bundled using AWS's Elastic Beanstalk (EB) and EC2 Container Registry (ECR) services to deploy to Brightcove's pre-existing environments. This project also gave us an opportunity to meet and share knowledge with LordNeckbeard of StackOverflow fame, king of all things ffmpeg, and owner one of the best screen names I've ever seen.

    Team members:
    • Brandon Aaskov,
    • Jesse Streb

  • Foundation Medicine, Inc

    November 2015 – May 2016

    Foundation Medicine is working on curing cancer by doing genomic analysis using next-generation sequencing technology. But to leverage that research in a meaningful way is no easy task. To solve this problem, we worked with the FMI team to create a product called Insights, a web tool to narrow down so many parameters (genes, age, diseases, etc.) into a visual representation of the data that can be comprehended quickly by the human eye. It would make Edward Tufte proud. The application has both a server side and client side piece, using node.js and React, respectively. We took the app from concept to beta where it was user-tested, all the way through launch. Using a lightweight Flux implementation, we were able to transition the project to their internal team with ease once the project was live.

    Team members:
    • Brandon Aaskov,
    • Jesse Streb

  • Smartvid.io

    June 2015 – November 2015

    Large, industrial-scale construction projects create massive amounts of video, both for training and insurance purposes. Smartvid.io capitalized on this by building a language-processing tool to automatically capture user-defined keywords from all uploaded video. The Rocket Insights team built out the first versions (alpha through launch) of the web application to ingest, encode, organize and manage these monolithic video libraries. Using a nested tree-like system for creating/managing smart tags, these large construction companies can now surface their content by searching for things spoken out loud in a video. Imagine in five years that some vendor needs to replace an HVAC unit in a large, modern structure... Now, they can easily surface that video and find out all about the original installation, the equipment, it's location, etc. That's just one of many, many applications of this cutting-edge technology that had never been used to solve such a large business problem.
